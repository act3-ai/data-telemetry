Labels

  • Simple key-value pairs that can be added to bottles and parts of bottles
  • Concept is the same as Kubernetes labels
  • Values are restricted in the same way as Kubernetes
  • Bottle authors can classify and identify sets of bottles with labels
  • Used to identify anything a user might want to use to select bottles
  • Also used on parts of bottles

Common uses for labels in ASCE Data Telemetry

  • Project that created it, e.g. project=COACH
  • Type of bottle, e.g. type=model problem=classification
  • (Hyper)Parameters used to generate the model, e.g. learning-rate=0.005 layers=7
  • Source code version, e.g. version=v0.2.4
  • A data set with both training and test sets contained within it as different parts, e.g. subset=train and subset=test
    This allows a user to restrict/limit what they download if only a few parts are required

Selectors

  • Used to restrict the set of bottles
  • Can make arbitrary groups of bottles by picking which selectors must match
  • Syntax is the same as Kubernetes label selectors
  • Composed of zero or more requirements
  • All requirements must match for a selector to match

Selectors in ASCE Data Telemetry

  • Using a comma( , ) to separate requirements semantically means AND, e.g. project=COACH,type=model
  • Whitespace within a requirement is insignificant
  • In some situations there is a temporary restriction that inequality requirements (7 and 8 below) only work when the value is an integer
  1. key = value and key == value both require that the value of key equal value
  2. key != value requires that the value of key not be value
  3. key in (value1, value2) requires that the value of key be either value1 or value2
  4. key notin (value1, value2) requires that the value of key be neither value1 or value2
  5. key requires that a label with key exist (the value of key is not checked)
  6. !key requires that a label with key not exist (the value of key is not checked)
  7. key < 5 requires that the value of key be less than 5
  8. key > 7 requires that the value of key be greater than 7
