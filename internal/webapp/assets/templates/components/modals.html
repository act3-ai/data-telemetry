Updating Bottle Schema Version

As mentioned in the Telemetry documentation, the Telemetry server has the ability to update a bottle's schema version automatically when it is received. This update to the metadata of the bottle will cause the creation of a new bottle in Telemetry with the correct schema.

To create the updated bottle in Telemetry, you will need to have the following:

  • Pull access for one of the OCI repositories where the bottle you want to update is stored
  • Push access for an OCI repository to push the bottle to
  • Your current ace-dt configuration includes this Telemetry server.
  • The latest version of the ACE Data Tool ace-dt installed.

Assuming the above requirements are met you may follow these instructions using the command-line:

  • Pull the outdated bottle: ace-dt bottle pull <registry>/<repository>/<name>:<tag>
  • Push the outdated bottle to your chosen registry: ace-dt bottle push <registry>/<repository>/<name>:<tag>
